Smart Contract API

This is the API documentation for NEO•ONE Smart Contracts. This includes all types and utilities available to import into your Smart Contract definition files from @neo-one/smart-contract.

Address

  • Buffer that represents a NEO address.
  • Stored as a script hash (Hash160) internally.
  • Methods:
    • isCaller: (Address) => boolean;
      • Verifies that the invocation was directly called AND approved by the given Address.
      • Smart contracts should invoke this function before taking transferring items for Addresses, like transferring tokens, that require the permission of the Address.
      • Example: 
        if (!Address.isCalled(inputAddress)) {
  return false;
}
    • isSender: (Address) => boolean;
      • Verifies that the Transaction was signed by the address.
      • In most cases, smart contracts should instead use. Address.isCaller.
      • Example: 
        if (!Address.isSender(address)) {
  return false;
}
  • Constructor:
    • from: (string) => Address;
      • Creates an Address from a literal string.
      • Accepts either a NEO address or a script hash.
      • Example: 
        const accountAddress = Address.from('ALq7AWrhAueN6mJNqk6FHJjnsEoPRytLdW');
const contractAddress = Address.from('​​​​​0xcef0c0fdcfe7838eff6ff104f9cdec2922297537​​​​​');
  • Example Type Usage: 
    @constant
public checkBalance(address: Address): Integer {
  // checks balance
};

Hash256

  • Buffer that represents a NEO 256 bit hash.
  • Examples of Hash256 include Block hashes, Transaction hashes, and Asset hashes.
  • Properties:
    • NEO
      • Hash256 of the NEO Asset.
      • Example: 
        sendAsset(fromAddress, toAddress, Hash256.NEO);
    • GAS
      • Hash256 of the GAS Asset.
      • Example: 
        sendAsset(fromAddress, toAddress, Hash256.GAS);
  • Constructor:
    • from: (string) => Hash256;
      • Creates a Hash256 from a literal string.
      • Example: 
        const transactionHash = Hash256.from('0xd6572a459b95d9136b7a713c5485ca709f9efa4f08f1c25dd792672d2bd75bfb')
  • Example Type Usage: 
    public sendAsset(from: Address, to: Address, asset: Hash256): boolean {
  // sends given asset represented by its Hash256 identifier.
};

PublicKey

  • Buffer that represents a public key.
  • Constructor:
    • from: (string) => PublicKey
      • Creates a PublicKey from a literal string.
      • Example: 
        const publicKey = PublicKey.from('02028a99826edc0c97d18e22b6932373d908d323aa7f92656a77ec26e8861699ef')
  • Example Type Usage: 
    public publicKeyToAddress(publicKey: PublicKey): Address {
  // converts public key to address
}

Fixed<Decimal extends number>

  • Integer which represents a number with the specified decimals.
  • Example Type Usage: 
    // 100.0000
public readonly tokenConversionRate: Fixed<4>: 100_0000;
// 10,000,000,000.00000000
public readonly remainingTokens: Fixed<8>: 10_000_000_000_00000000;

Integer

  • Integer that represents a number with 0 decimals.
  • Shorthand for Fixed<0>.
  • Example Type Usage: 
    public readonly icoDurationSeconds: Integer = 86400;

Fixed8

  • Integer that represents a number with 8 decimals.
  • Shorthand for Fixed<8>.
  • Example Type Usage: 
    public gasBalance(address: Address): Fixed8 {
  // returns GAS balance of address
}

TransactionType

  • enum of all Transaction Types.
    • Miner = 0x00
      • First Transaction in each block which contains the Block rewards for the consensus node that produced the Block.
      • Example: 
        if(transaction.type === TransactionType.Miner) {
  // do something
}
    • Issue = 0x01
      • Issues new currency of a first-class Asset.
      • Example: 
        if(transaction.type === TransactionType.Issue) {
  // do something
}
    • Claim = 0x02
      • Claims GAS for a set of spent Outputs.
      • Example: 
         if(transaction.type === TransactionType.Claim) {
  // do something
}
    • Contract = 0x80
      • Transfers first class Assets.
      • Example: 
         if(transaction.type === TransactionType.Contract) {
  // do something
}
    • State = 0x90
      • State Transaction type.
      • Example: 
         if(transaction.type === TransactionType.State) {
  // do something
}
    • Publish = 0xd0
      • Registers a new Contract
      • Example: 
         if(transaction.type === TransactionType.Publish) {
  // do something
}
    • Invocation = 0xd1
      • Runs a script in the NEO-VM.
      • Example: 
         if(transaction.type === TransactionType.Invocation) {
  // do something
}

BufferAttributeUsage

  • Attribute usage flag indicating the data is an arbitrary Buffer.
    • AttributeUsage.DescriptionUrl
    • AttributeUsage.Description
    • AttributeUsage.Remark
    • AttributeUsage.Remark1 ... AttributeUsage.Remark15
  • See BufferAttribute.

PublicKeyAttributeUsage

  • Attribute usage flag indicating the data is a PublicKey.
    • AttributeUsage.ECDH02
    • AttributeUsage.ECDH03
  • See PublicKeyAttribute

AddressAttributeUsage

  • Attribute usage flag indicating the data is an Address.
    • AttributeUsage.Script
  • See AddressAttribute

Hash256AttributeUsage

  • Attribute usage flag indicating the data is a Hash256.
    • AttributeUsage.ContractHash
    • AttributeUsage.Vote
    • AttributeUsage.Hash1 ... AttributeUsage.Hash15

AttributeBase

  • Base interface for Attributes.
  • Properties:
    • usage: AttributeUsage
      • Attribute usage flag indicating the type of the data
    • data: Buffer
      • The raw data stored in the Attribute

BufferAttribute

  • Attribute whose data is an arbitrary Buffer.
  • Extension of AttributeBase.
  • Properties:
    • usage: BufferAttributeUsage
    • data: Buffer

PublicKeyAttribute

  • Attribute whose data is a PublicKey.
  • Extension of AttributeBase.
  • Properties:
    • usage: PublicKeyAttributeUsage
    • data: PublicKey

AddressAttribute

  • Attribute whose data is an Address.
  • Extension of AttributeBase.
  • Properties:
    • usage: AddressAttributeUsage
    • data: Address

Hash256Attribute

  • Attribute whose data is a Hash256.
  • Extension of AttributeBase.
  • Properties:
    • usage: Hash256AttributeUsage
    • data: Hash256

Attribute

  • Attributes are used to store additional data on Transactions.
    • BufferAttribute
    • PublicKeyAttribute
    • AddressAttribute
    • Hash256Attribute
  • Most attributes are used to store arbitrary data, whereas some, like AddressAttribute, have specific uses in the NEO protocol.

Output

  • Outputs represent the destination Address and amount transferred of a given Asset.
  • The sum of the unspent Outputs of an Address represent the total balance of an Address.
  • Properties:
    • address: Address
      • Destination Address
    • asset: Hash256
      • Hash of the Asset that was tranferred.
    • value: Fixed8
      • Amount transferred.
  • Example: 
    private checkNEOBalance(address: Address, outputs: ReadonlyArray<Output>): Fixed8 {
  let amount = 0;
  for (const output of outputs) {
    if (output.address === address) {
      if (output.asset === Hash256.NEO) {
        amount += output.value;
      }
    }
  }

  return amount;
};

Input

  • Inputs are a reference to an Output of a Transaction that has been persisted to the blockchain.
  • The sum of the value of the referenced Outputs is the total amount transferred in the Transaction.
  • Properties:
    • hash: Hash256
      • Hash of this Transaction.
    • index: Integer
      • Output index within the Transaction this input references.

TransactionBase

  • Base interface for all Transactions.
  • Properties:
    • hash: Hash256
      • Hash256 of the Transaction.
    • type: TransactionType
      • Type of the Transaction.
    • attributes: Attribute[ ]
      • Attributes attached to the Transaction.
    • outputs: Output[ ]
      • Outputs of the Transaction.
    • inputs: Input[ ]
      • Inputs of the Transaction.
    • references: Output[ ]
      • Corresponding Outputs for the Inputs of the Transaction.
    • unspentOutputs: Output[ ]
      • Outputs which have not been spent.

MinerTransaction

  • Extension of TransactionBase
  • First Transaction in each block which contains the Block rewards for the consensus node which produced the Block.
  • Properties:
    • type: TransactionType.Miner

IssueTransaction

  • Extension of TransactionBase
  • Issues new currency of a first-class Asset.
  • Properties:
    • type: TransactionType.Issue

ClaimTransaction

  • Extension of TransactionBase
  • Claims GAS for a set of spent Outputs.
  • Properties:
    • type: TransactionType.Claim

ContractTransaction

  • Extension of TransactionBase
  • Transaction that transfers first class Assets.
  • Properties:
    • type: TransactionType.Contract

StateTransaction

  • Extension of TransactionBase
  • Properties:
    • type: TransactionType.State

InvocationTransaction

  • Extension of TransactionBase
  • Transaction which runs a script in the NEO-VM.
  • Properties:
    • type: TransactionType.Invocation
    • script: Buffer
      • Code that was executed in the NEO-VM.

Transaction

  • Transactions are persisted to the blockchain and represent various functionality like transferring first class Assets or executing smart contracts.
    • MinerTransaction
    • IssueTransaction
    • ClaimTransaction
    • ContractTransaction
    • StateTransaction
    • InvocationTransaction
  • Constructor:
    • for: (Hash256) => Transaction
  • Example: 
    const transactionHash = Hash256.from('0xd6572a459b95d9136b7a713c5485ca709f9efa4f08f1c25dd792672d2bd75bfb');
const transaction = Transaction.for(transactionHash);
const transactionOutputs = transaction.outputs;

Account

  • Object which stores an Address and a method for checking its balances.
  • Properties:
    • address: Address
      • Address of the Account.
  • Methods:
    • getBalance: (Hash256) => Fixed8
      • Retrieves the balance of a first class Asset for the Account's Address.
  • Constructor:
    • for: (Address) => Account
      • Returns an Account for the specified Address.
  • Example: 
    const address = Address.from('ALq7AWrhAueN6mJNqk6FHJjnsEoPRytLdW');
const account = Account.for(address);
const neoBalance = account.getBalance(Hash256.NEO);

AssetType

  • enum of all possible types for Assets.
    • Credit = 0x40
    • Duty = 0x80
    • Governing = 0x00
      • Reserved for the NEO Asset.
    • Utility = 0x01
      • Reserved for the GAS Asset.
    • Currency = 0x08
    • Share = 0x90
    • Invoice = 0x98
    • Token = 0x60

Asset

  • Attributes of a first-class Asset.
  • Smart Contract authors will typically only interact with the NEO and GAS Assets.
  • Properties:
    • hash: Hash256
      • Hash256 of the Asset.
    • type: AssetType
      • Type of the Asset which denotes its function.
    • amount: Fixed8
      • Total possible supply of the Asset.
    • available: Fixed8
      • Amount currently available of the Asset.
    • precision: Integer
      • Precision (number of decimal places) of the Asset.
    • owner: PublicKey
      • Owner of the Asset.
    • admin: Address
      • Admin of the Asset.
    • issuer: Address
      • Issuer of the Asset.
  • Constructor:
    • for: (hash: Hash256) => Asset
      • Returns an Asset for the specified hash.
  • Example: 
    const neoAsset = Asset.for(Hash256.NEO);
const neoAmount = neoAsset.amount;

Contract

  • Attributes of a smart contract deployed to the blockchain.
  • Properties:
    • script: Buffer
      • Contract code.
    • payable: boolean
      • Flag that indicates if the Contract supports receiving Assets and NEP-5 tokens.
  • Constructor:
    • for: (address: Address) => Contract | undefined
      • Returns a Contract for the specified Address.
      • Returns undefined if a Contract does not exist at the specified Address.
  • Example: 
    const contractAddress = Address.from('​​​​​0xcef0c0fdcfe7838eff6ff104f9cdec2922297537​​​​​');
const contract = Contract.for(contractAddress);
const contractScript = contract.script;

Header

  • Attributes of a Block persisted to the blockchain.
  • Header includes all information except for the Transactions.
  • Properties:
    • hash: Hash256
      • Block hash.
    • version: Integer
      • NEO blockchain version.
    • previousHash: Hash256
      • hash of the previous Block.
    • index: Integer
      • Block index.
    • merkleRoot: Hash256
    • time: Integer
      • Block time persisted.
    • nextConsensus: Address
      • Next consensus Address.
  • Constructor:
    • for: (Hash256 | Integer) => Header
      • Accepts either a Hash256 representing a Block hash or the index of the Block.
      • Returns a Header for the specified block hash or index.
  • Example: 
    const blockHash = Hash256.from('0xd6572a459b95d9136b7a713c5485ca709f9efa4f08f1c25dd792672d2bd75bfb');
const header = Header.for(blockHash);
const nextConsensusAddress = header.nextConsensus;

Block

  • Attributes of a Block persisted to the blockchain.
  • Extension of Header.
  • Properties:
    • transactions: Transaction[]
      • Transactions included in the Block.
  • Constructor:
    • for: (Hash256 | Integer) => Block;
      • Accepts either a Hash256 representing a Block hash or the index of the Block.
      • Returns a Block for the specified block hash or index.
  • Example: 
    const genesisBlock = Block.for(0);

SerializableKeySingle

  • Possible single key types for MapStorage and SetStorage.
    • number
    • string
    • boolean
    • Buffer

SerializableKey

  • Possible key types for MapStorage and SetStorage. This includes single key values (SerializableKeySingle) and collections of single key values. Supports collections of up to four single key values.
    • SK
    • [SK, SK]
    • [SK, SK, SK]
    • [SK, SK, SK, SK]

SerlializableValue

  • Possible value types for ArrayStorage and MapStorage.
    • undefined
    • null
    • number
    • string
    • boolean
    • Buffer
    • SerializableValue Array
    • SerializableValue Map
    • SerializableValue Set

ArrayStorage

  • Persistent smart contract array storage.
  • Only usable as a SmartContract property.
  • Properties:
    • length: number
      • The length of the array. This is a number one higher than the index of last element defined in the array.
  • Methods:
    • forEach: (callback: (value: T, idx: number) => void) => void
      • Executes a provided callback function once per each value in storage.
    • push: (...items: T[ ]) => number
      • Appends new elements to storage and returns the new length of the array.
    • pop: () => T | undefined;
      • Removes the last element from an array and returns it.
  • Constructor:
    • for<T extends SerializableValue>(): ArrayStorage
      • Constructs a new ArrayStorage instance. Only usable as a SmartContract property.
  • Example: 
    class MySmartContract extends SmartContract {
  private readonly pendingAddresses = ArrayStorage.for<Address>();

    public addPendingAddress(address: Address): void {
      this.pendingAddresses.push(address);
    }
}

MapStorage

  • Persistent smart contract map storage.
  • Only usable as a SmartContract property.
  • Methods:
    • forEach(callback: (value: V, key: K) => void) => void
      • Executes a provided function once per each key/value pair in storage.
    • get: (key: K) => V | undefined;
      • Returns the value associated with the given key from storage.
    • has: (key: K) => boolean;
      • Returns true when the storage contains an element with the given key. False if no such element exists in the storage.
    • delete: (key: K) => boolean;
      • Removes the element associated with the given key from storage.
      • Returns true if the element existed and has been removed or false if the element does not exist.
    • set: (key: K, value: V) => MapStorage<K, V>
      • Adds or updates an element with a specified key and value in storage.
      • Returns the MapStorage object.
  • Constructor:
    • for<T extends SerializableKey, V extends SerializableValue>(): MapStorage<K, V>
      • Constructs a new MapStorage instance. Only usable as a SmartContract property.
  • Example: 
    class MyToken extends SmartContract {
  private readonly balances = MapStorage.for<Address, Fixed<8>>();

  public transfer(
    from: Address,
    to: Address,
    amount: Fixed<8>,
  ): boolean {
    const fromBalance = this.balances.get(from);
    const toBalance = this.balances.get(to);
    this.balances.set(from, fromBalance - amount);
    this.balances.set(to, toBalance + amount);

    return true
  }
}

SetStorage

  • Persistent smart contract set storage.
  • Only usable as a SmartContract property.
  • Methods:
    • forEach: (callback: (value: V) => void) => void
      • Executes a provided function once per each value in storage.
    • has: (value: V) => boolean
      • Returns a boolean indicating whether a given value exists in the storage.
    • delete: (value: V) => boolean
      • Removes the specified value from storage.
      • Returns true if the value existed in storage and was removed.
    • add: (value: V) => SetStorage
      • Adds a given value to the storage.
      • Returns the SetStorage object.
  • Constructor:
    • for<V extends SerializableKey)>(): SetStorage
      • Constructs a new SetStorage instance. Only usable as a SmartContract property.
  • Example: 
    class ICO extends SmartContract {
  private readonly whitelistedAddresses = SetStorage.for<Address>();

  public isWhiteListed(address: Address): boolean {
    return this.whitelistedAddresses.has(address);
  }
}

Blockchain

  • Information about the current state of the blockchain and the current execution.
  • Properties:
    • currentBlockTime: number
      • Time of the current Block.
      • During execution, this is the timestamp of the Block that this Transaction will be included in.
      • During verification, this is the timestamp of the latest Block.
      • currentHeight: number
        • Index of the latest Block persisted to the blockchain.
      • currentTransaction: InvocationTransaction
        • InvocationTransaction this smart contract is executed in.

Deploy

  • Inject values at deployment time.
  • Can only be used for default constructor parameters.
  • Properties:
    • senderAddress: Address
      • Use sender Address for the constructor parameter.
  • Example: 
    import { Address, Deploy, SmartContract } from '@neo-one/smart-contract'

class Token extends SmartContract {
  public constructor(public readonly owner: Address = Deploy.senderAddress) {
    super();
  }
}

ContractProperties

  • Required information for the deployment of a SmartContract.
  • Must be included as a public readonly property named properties on all SmartContracts
  • Properties:
    • codeVersion: string
    • author: string
    • email: string
    • description: string
  • Example: 
    import { SmartContract } from '@neo-one/smart-contract'

export class MySmartContract extends SmartContract {
  public readonly properties = {
    codeVersion: '1.0',
    author: 'my name',
    email: 'myName@email.io',
    description: 'My Contract',
  };

SmartContract

  • Marks a class as a SmartContract.
  • Smart Contracts written with NEO•ONE are classes which extend this SmartContract base class.
  • Properties:
    • properties: ContractProperties
      • codeVersion: string
      • author: string
      • email: string
      • description: string
    • address: Address
      • Address of the SmartContract.
    • processedTransactions: SetStorage
      • Stores Transaction hashes that have sent or received native assets.
      • Used to enforce that a Transaction with native assets is only ever processed appropriately.
    • deployed: true
      • Property primarily used internally to validate that the smart contract is deployed only once.
  • Methods:
    • refundAssets: (transactionHash: Hash256) => boolean
      • Method automatically added for refunding native assets.

LinkedSmartContract

  • An object representing a statically linked contract.
  • Constructor:
    • for: <T extends SmartContract> => T | never;
      • Returns an object representing a statically linked contract T.
      • T is checked for validity and LinkedSmartContract.for will report an error during compilation if the interface is invalid.
  • Example: 
    import { Token } from './Token';

const contract = linkedSmartContract.for<Token>();
const from = Address.from('ALfnhLg7rUyL6Jr98bzzoxz5J7m64fbR4s');
const to = Address.from('AVf4UGKevVrMR1j3UkPsuoYKSC4ocoAkKx');

contract.transfer(from, to, 10);

Hashable

  • Hashable types usable by the Crypto utilities.
    • number
    • string
    • boolean
    • Buffer

Crypto

  • Collection of some common cryptography utilities
    • sha1: (Hashable) => Buffer
    • sha256: (Hashable) => Buffer
    • hash160: (Hashable) => Buffer
    • hash256: (Hashable) => Buffer

constant

  • Tag to mark a SmartContract method as not modifying storage.
  • Use above a method as @constant.
  • Example:
@constant
public checkBalance(address: Address): Integer {
  // checks balance, does not alter storage
}
