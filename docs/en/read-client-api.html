Read Client API

This is the API documentation for the NEO•ONE Read Client. The Read Client is accessed through the read method on the Client and is primarily used to retrieve information about the blockchain.

const readClient = await client.read(networkName);

Methods Reference

getAccount(AddressString): Promise<Account>

  • Used to obtain an Account from an input AddressString.

getAsset(Hash256String): Promise<Asset>

  • Used to obtain an Asset identified by its given Hash256String id.

getBlock(number | Hash256String, GetOptions?): Promise<Block>

  • Retrieve a Block identified by its index or Hash256String identifier.
  • Optionally pass in a GetOptions object as the second argument to set a timeout.

iterBlocks(BlockFilter?): AsyncIterable<Block>

  • Returns a stream of Blocks as they become available.
  • Filter the Blocks by passing in an optional BlockFilter.

getBestBlockHash(): Promise<Hash256String>

  • Returns the Hash256String identifier for the most recent available Block.

getBlockCount(): Promise<number>

  • Returns the index of the most recent available Block.

getContract(AddressString): Promise<Contract>

  • Returns a Contract associated with a given AddressString

getMemPool(): Promise<ReadonlyArray<Hash256String>>

  • Returns the current Mem Pool.
  • The Mem Pool is the pool of Transactions waiting to be added into a Block and is stored as an array of Transaction Hash256String identifiers.

getTransaction(Hash256String): Promise<Transaction>

  • Returns the Transaction associated with the given Hash256String identifier.

getOutput(Input): Promise<Output>

  • Returns the Output referenced by the given Input.

getConnectedPeers(): Promise<ReadonlyArray<Peer>>

  • Returns an array of the connected Nodes.

smartContract<T extends ReadSmartContract>(ReadSmartContractDefinition): T

  • Returns a ReadSmartContract for a given ReadSmartContractDefinition.

getStorage(AddressString, BufferString): Promise<StorageItem>

  • Returns the persistent storage for the Contract associated with the given AddressString.
  • The StorageItem returned is the one associated with the given key represented as a BufferString.

iterStorage(AddressString): AsyncIterable<StorageItem>

  • Returns a stream of StorageItems in persistent storage for the Contract associated with the given AddressString as they become available.
