Key Store API

The NEO•ONE Key Store API is used for directly managing Accounts and Wallets for the Client. The keystore is accessed through the Client's providers.

const keystore = client.providers.memory.keystore;

Methods

  • getCurrentAccount
  • getAccounts
  • sign
  • selectAccount
  • updateAccountName
  • getWallet
  • getWallets$
  • addAccount
  • deleteAccount
  • unlockWallet

Methods Reference

getCurrentAccount(): UserAccount

  • Returns the UserAccount currently selected by the keystore. This is the default UserAccount for all operations.
  • For most cases, use getCurrentAccount directly from the Client
  • Example: 
    const keystore = client.providers.memory.keystore;

const currentAccount = keystore.getCurrentAccount();
const currentAccountName = currentAccount.name;

getAccounts(): ReadonlyArray<UserAccount>

  • Returns all UserAccounts stored in the keystore.
  • For most cases, use getAccounts directly from the Client
  • Example: 
    const keystore = client.providers.memory.keystore;

const accountNames = keystore.getAccounts().map((account) => account.name);

sign({UserAccountID, string}): Promise<Witness>

  • Creates a signature using the provided UserAccountID's private key and the provided string message. Returns the created Witness.
  • In most cases, signatures are handled automatically by methods in one of the higher level APIs.

selectAccount(UserAccountID?): Promise<void>

  • Sets the UserAccount associated with the provided UserAccountID to be the current account used as the default for all operations.
  • If no input is provided, a new account will be created and selected.
  • For most cases use selectAccount directly from the Client.
  • Example: 
    const keystore = client.providers.memory.keystore;

await keystore.selectAccount();
// currentAccount will be the account newly created by selectAccount
const currentAccount = keystore.getCurrentAccount();

updateAccountName(UpdateAccountNameOptions): Promise

<void>

  • Updates the name associated with the account identified by the provided UpdateAccountNameOptions for a wallet that is stored in this keystore.
  • For most cases use updateAccountName directly from the Client.
  • Example: 
    const keystore = client.providers.memory.keystore;

const account = keystore.getAccounts()[0];
await keystore.updateAccountName({
  id: account.id,
  name: 'new-account-name',
});

await keystore.selectAccount(account.id);
// prints 'new-account-name' to the console
console.log(client.getCurrentAccount().name);

getWallet(UserAccountID): LocalWallet

  • Returns the LocalWallet associated with the provided UserAccountID that is stored in the keystore.
  • Example: 
    function isWalletLocked(accountID: UserAccountID, keystore: LocalKeyStore): boolean {
  const wallet = keystore.getWallet(accountID);

  return wallet.type === 'locked';
}

getWallets$(UserAccountID): Observable<LocalWallet | undefined>

  • Returns an Observable which emits the LocalWallet associated with the given UserAccountID if it exists in the keystore. Emits undefined otherwise.

addAccount({network: NetworkType, privateKey?: BufferString, name?: string, password?: string, nep2?: string}): Promise<LocalWallet>

  • Adds a LocalWallet to the keystore.
  • Requires either a privateKey or a password with its associated NEP-2 key.
  • Example: 
    const keystore = client.providers.memory.keystore;

const wallet = await keystore.addAccount({
  network: 'private',
  privateKey: createPrivateKey(),
});
const walletAddress = wallet.account.id.address;

deleteAccount(UserAccountID): Promise<void>

  • Removes LocalWallet associated with the given UserAccountID from the keystore.
  • For most cases use deleteAccount directly from the Client.
  • Example: 
    const keystore = client.providers.memory.keystore;

const userAccount = keystore.getAccounts()[0];
// prints 'Aha!' one time.
for (const account of keystore.getAccounts()) {
  if (account.id === userAccount.id) {
    console.log('Aha!')
  }

await keystore.deleteAccount(userAccount.id);

// prints nothing.
for (const account of keystore.getAccounts()) {
  if (account.id === userAccount.id) {
    console.log('Aha!')
  }
}

unlockWallet({id: UserAccountID, password: string}): Promise<void>

  • Unlocks the LocalWallet associated with the provided UserAccountID stored in the keystore using the provided password.
  • Unlocking a LocalWallet uses the password with the stored NEP-2 key to decrypt and store the private key.
  • Example: 
    const keystore = client.providers.memory.keystore;

const { password, userAccountID } = getDataFromUser();
await unlockWallet({id: userAccountID, password});

lockWalet(id: UserAccountID): Promise<void>

  • Locks the LocalWallet associated with the provided UserAccountID stored in the keystore.
  • Locking a LocalWallet removes the stored private key and password data. Keeps only the NEP-2 key.
  • Example: 
    async function lockAllWallets(keystore: LocalKeyStore: Promise<void> {
  const accounts = keystore.getAccounts();

  await Promise.all(accounts.map((account) => lockWallet(account.id)));
}
